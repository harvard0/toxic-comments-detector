我们在脚本中采用的 4 种核心清洗方法：

# 不可见字符清理：
通过正则去除了大模型生成时偶尔携带的零宽字符（如 \u200b, \u202d 等），防止模型 Tokenizer 报错。

# AI 痕迹剥离（内容级修改，不删除整条）：
利用正则表达式，自动去除了大模型特有的“开场白”（如“好的，为您生成...”）和“刻板总结语”（如“综上所述...”）。

# 极端长度过滤：
直接剔除了清洗后字数不足 5 个字的无意义短评，以及超过 300 个字的过度发散长评。

# 双重强力去重：

## 完全精确去重：
剔除一字不差的重复样本。

## 归一化去重：
去除所有标点符号和空格后进行比对，剔除核心文字完全一样但标点符号不同的“伪原创”数据。
